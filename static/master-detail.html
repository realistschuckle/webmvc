Master-Detail Relationships

This page presents a master/detail relationship allowable through the links embedded in the page.

If you browse the source of the page, you will see that it contains a link tag with described relationships for the client (this Web application) to use to work with an unloaded subentites state transition.

This resource also loads an external resource named "subentity-registry.js" which acts as the application's monitor for the subentities that this resource can contain. 

<link rel="unloaded-subentities-uri-template" href="{/entity}">
<script src="subentity-registry.js"></script>

So, with all of this in this page, we have two state transitions:

  • Follow the "Back" link at the top of the page
  • Either initiate a transition (or as an application, automatically follow the transition) templated in the link tag in the head.
For demonstrative purposes, this application automatically makes the request on your behalf to transition the application to navigate to the subentities represented by the URN in the fragment of the URL.
